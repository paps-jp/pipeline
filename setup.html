セットアップ

制御プレーン 1 ホスト + ワーカー 1 ホスト以上、 という構成を 想定。 小規模では 同一ホストで 両方動かしても 構わない。

このページの内容

  • 必要要件
  • インストール (制御プレーン)
  • 初回起動
  • Web UI ビルド
  • 最初のワークロード
  • ワーカーをホストに追加
  • systemd 常駐化
  • 動作確認

必要要件

項目バージョン備考
Python3.12 以上制御プレーン / ワーカー 共通
Node.js18 以上Web UI ビルド時のみ (= 配布済 build を使うなら不要)
OSLinux 推奨制御プレーン。 Mac / WSL でも 開発可。
SQLite3.35 以上ALTER TABLE DROP COLUMN を使うため
GPU ホスト任意NVIDIA GPU + ドライバ + nvidia-smi (= GPU メトリクスを取る場合)

インストール (制御プレーン)

git clone https://github.com/paps-jp/pipeline
cd pipeline

# venv 作成
python3.12 -m venv .venv
source .venv/bin/activate     # Windows: .venv\Scripts\activate

# パッケージインストール (editable)
pip install -e .

これで pipeline コマンドが venv の中に入る。

初回起動

SQLite DB は 初回起動時に 自動作成される。

pipeline run \
  --host 0.0.0.0 \
  --port 8001 \
  --db-url sqlite:///./pipeline.db \
  --log-level info

ブラウザで http://localhost:8001 を開く。 ダッシュボードが 表示されれば OK。

初回は ワークロード ゼロ。 「実行中」 「最近の失敗」 「キュー深さ」 の 3 つの パネルは 全て 空状態で表示される。 これは 正常。

Web UI ビルド

repo にビルド済バンドルは 含まれないため、 初回は ビルドが必要。

cd web
npm install
npm run build
# → ../pipeline/web/static/ に index.html + assets/ が出力される

その後 pipeline run … を 再起動すると FastAPI が 新 UI を 配信する。

開発時 (= ホットリロード) は npm run dev。 localhost:5173 が 立ち上がり /api は localhost:8001 へ proxy される。

最初のワークロード

repo に plugins/echo_sample/ が同梱されている。 これを 使って 動作確認できる。

  1. Web UI → ワークロード タブ → 新規作成
  2. 基本情報:
    • slug: echo-test (= URL に使われる識別子)
    • 表示名: Echo テスト
  3. 実行モード: python_module を選択
  4. プラグイン: echo-sample を選択
  5. モジュール: main を選択
  6. 設定:
    • prefix: [echo] (デフォルト)
    • sleep_secs: 0
    • fail_pk_substr: 空文字
  7. 作成 ボタン → ワークロード 一覧に echo-test が追加される
  8. その行の 📨 投入 アイコン → pk に hello と入れて 投入
  9. 同じ行の 履歴 アイコン → runs drawer で 結果が緑色 (= success) で 表示される

1 件 実行できれば 制御プレーンは 正常。 次は ワーカーを 別ホストに 追加する。

ワーカーをホストに追加

方法 1: 手動

ワーカーをデプロイしたい ホストに SSH ログインして、 同じ手順で venv + pip install を 行う。 その後:

pipeline worker \
  --control-url http://<control-host>:8001 \
  --worker-id worker-host-a \
  --tags gpu \
  --log-level info

制御プレーンの Web UI → ワーカー タブで このホストが active と表示される。

方法 2: Deploy タブから配信

運用フェーズに 入ったら 制御プレーンから コードを 配信できる。

  1. Web UI → Deploy タブ
  2. 「配信先ホスト」 で SSH 接続先 (= IP / port / user) を 登録
  3. 「配信パス」 で 何を どこへ コピーするか + 配信後 実行コマンド + service コマンドを 登録
  4. Deploy now ボタン → 一括で rsync + systemctl restart

詳細は マニュアルの Deploy セクション。

systemd 常駐化

制御プレーン

# /etc/systemd/system/pipeline-control.service
[Unit]
Description=Pipeline control plane
After=network.target

[Service]
Type=simple
User=youruser
WorkingDirectory=/opt/pipeline
ExecStart=/opt/pipeline/.venv/bin/pipeline run \
    --host 0.0.0.0 --port 8001 \
    --db-url sqlite:////opt/pipeline/data/pipeline.db \
    --log-level info
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target

ワーカー

# /etc/systemd/system/pipeline-worker.service
[Unit]
Description=Pipeline worker
After=network.target

[Service]
Type=simple
User=youruser
WorkingDirectory=/opt/pipeline
ExecStart=/opt/pipeline/.venv/bin/pipeline worker \
    --control-url http://<control-host>:8001 \
    --worker-id %H \
    --tags gpu \
    --log-level info
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now pipeline-control     # 制御プレーン側
sudo systemctl enable --now pipeline-worker      # 各ワーカー側

動作確認

curl で API を叩いて 状態を 確認:

curl http://localhost:8001/api/v1/status
# → {"version":"0.1.0","mode":"prod","db_url":"sqlite:///./pipeline.db","now":"..."}

curl http://localhost:8001/api/v1/workers
# → {"workers":[{"id":"worker-host-a","host":"...","state":"active",...}],"total":1}

curl http://localhost:8001/api/v1/dashboard/overview
# → {"running":[...],"recent_failures":[...],"queue_depths":[...]}

続いて 詳細な 使い方は マニュアル へ。
