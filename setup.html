セットアップ

制御プレーン1ホスト + ワーカー1ホスト以上、という構成を想定。 小規模では同一ホストで両方動かしても構わない。

必要要件

項目バージョン備考
Python3.12以上制御プレーン / ワーカー共通
Node.js18以上Web UIビルド時のみ (= 配布済buildを使うなら不要)
OSLinux推奨制御プレーン。Mac / WSLでも開発可。
SQLite3.35以上ALTER TABLE DROP COLUMNを使うため
GPUホスト任意NVIDIA GPU + ドライバ + nvidia-smi (= GPUメトリクスを取る場合)

インストール (制御プレーン)

git clone https://github.com/paps-jp/pipeline
cd pipeline

# venv作成
python3.12 -m venv .venv
source .venv/bin/activate     # Windows: .venv\Scripts\activate

# パッケージインストール (editable)
pip install -e .

これで pipeline コマンドがvenvの中に入る。

初回起動

SQLite DBは初回起動時に自動作成される。

pipeline run \
  --host 0.0.0.0 \
  --port 8001 \
  --db-url sqlite:///./pipeline.db \
  --log-level info

ブラウザで http://localhost:8001 を開く。ダッシュボードが表示されればOK。

初回はワークロードゼロ。 「実行中」「最近の失敗」「キュー深さ」の3つの パネルは全て空状態で表示される。これは正常。

Web UIビルド

repoにビルド済バンドルは含まれないため、初回はビルドが必要。

cd web
npm install
npm run build
# → ../pipeline/web/static/ にindex.html + assets/ が出力される

その後 pipeline run … を再起動するとFastAPIが新UIを配信する。

開発時 (= ホットリロード) は npm run dev。 localhost:5173 が立ち上がり /api は localhost:8001 へproxyされる。

最初のワークロード

repoに plugins/echo_sample/ が同梱されている。これを使って動作確認できる。

  1. Web UI → ワークロード タブ → 新規作成
  2. 基本情報:
    • slug: echo-test (= URLに使われる識別子)
    • 表示名: Echoテスト
  3. 実行モード: python_module を選択
  4. プラグイン: echo-sample を選択
  5. モジュール: main を選択
  6. 設定:
    • prefix: [echo] (デフォルト)
    • sleep_secs: 0
    • fail_pk_substr: 空文字
  7. 作成 ボタン → ワークロード一覧に echo-test が追加される
  8. その行の 📨 投入 アイコン → pkに hello と入れて 投入
  9. 同じ行の 履歴 アイコン → runs drawerで結果が緑色 (= success) で表示される

1件実行できれば制御プレーンは正常。次はワーカーを別ホストに追加する。

ワーカーをホストに追加

方法1: 手動

ワーカーをデプロイしたいホストにSSHログインして、同じ手順でvenv + pip installを行う。その後:

pipeline worker \
  --control-url http://<control-host>:8001 \
  --worker-id worker-host-a \
  --tags gpu \
  --log-level info

制御プレーンのWeb UI → ワーカー タブでこのホストが active と表示される。

方法2: Deployタブから配信

運用フェーズに入ったら制御プレーンからコードを配信できる。

  1. Web UI → Deploy タブ
  2. 「配信先ホスト」でSSH接続先 (= IP / port / user) を登録
  3. 「配信パス」で何をどこへコピーするか + 配信後実行コマンド + serviceコマンドを登録
  4. Deploy now ボタン → 一括でrsync + systemctl restart

詳細は マニュアルのDeployセクション。

systemd常駐化

制御プレーン

# /etc/systemd/system/pipeline-control.service
[Unit]
Description=Pipeline control plane
After=network.target

[Service]
Type=simple
User=youruser
WorkingDirectory=/opt/pipeline
ExecStart=/opt/pipeline/.venv/bin/pipeline run \
    --host 0.0.0.0 --port 8001 \
    --db-url sqlite:////opt/pipeline/data/pipeline.db \
    --log-level info
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target

ワーカー

# /etc/systemd/system/pipeline-worker.service
[Unit]
Description=Pipeline worker
After=network.target

[Service]
Type=simple
User=youruser
WorkingDirectory=/opt/pipeline
ExecStart=/opt/pipeline/.venv/bin/pipeline worker \
    --control-url http://<control-host>:8001 \
    --worker-id %H \
    --tags gpu \
    --log-level info
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now pipeline-control     # 制御プレーン側
sudo systemctl enable --now pipeline-worker      # 各ワーカー側

動作確認

curlでAPIを叩いて状態を確認:

curl http://localhost:8001/api/v1/status
# → {"version":"0.1.0","mode":"prod","db_url":"sqlite:///./pipeline.db","now":"..."}

curl http://localhost:8001/api/v1/workers
# → {"workers":[{"id":"worker-host-a","host":"...","state":"active",...}],"total":1}

curl http://localhost:8001/api/v1/dashboard/overview
# → {"running":[...],"recent_failures":[...],"queue_depths":[...]}
